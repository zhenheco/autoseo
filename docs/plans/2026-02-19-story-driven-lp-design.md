# Story-Driven Landing Page Redesign

## Overview

將 1WaySEO 首頁從功能導向改寫為故事驅動的旅程式 Landing Page。

## 設計決策

| 面向     | 決定                                |
| -------- | ----------------------------------- |
| 範圍     | 改寫現有首頁（非新增頁面）          |
| 主要受眾 | 行銷主管 + 電商老闆                 |
| 敘事結構 | 旅程式（痛點 → 掙扎 → 轉變 → 成果） |
| 視覺調性 | 溫暖共感型（柔和色調、感性語氣）    |
| CTA 策略 | 單一目標：免費試用 3 篇             |
| 故事方案 | 單一主角（行銷主管小美）貫穿全頁    |

## LP 結構（8 個 Section）

### Section 1: Hero — 共感開場

- 場景：晚上 11 點，小美還在改第 7 篇文章
- 轉折句：「有的。而且比你想像的簡單很多。」
- CTA：「免費體驗看看」
- 視覺：深色調營造深夜辦公室氛圍，文字逐行淡入

### Section 2: 痛點深化 — 「你試過的那些方法」

- 三張卡片：請外包寫手（太貴）/ 自己硬寫（太累）/ ChatGPT 拼湊（品質差）
- 收尾：「不是你不夠努力，是方法不對。」
- 視覺：淡紅橘色調，簡單 icon

### Section 3: 轉折 — 「直到那個星期一早上」

- 同事推薦 → 半信半疑試用 → 愣住了
- 用小美視角描述功能：「它自己去查了競品」「數據是真的」「圖片都生好了」
- 視覺：從深色漸變到暖白色，象徵天亮了

### Section 4: 成果展現 — 「三個月後的小美」

- 三個生活片段取代數據統計
- 片段 1：準時下班，一個下午排好整週文章
- 片段 2：成本從 6 萬降到 4,500
- 片段 3：3 篇文章排進 Google 第一頁
- 視覺：溫暖日光色調，數字柔和浮在角落

### Section 5: 社會證明 — 「不只小美這樣」

- 對話截圖風格（LINE 聊天感）取代制式 testimonial
- 三段對話：電商老闆阿明 / 行銷經理 Jenny / SEO 顧問阿凱
- 底部低調數據：500+ 企業、50,000+ 篇、4.9 滿意度
- 視覺：圓角氣泡卡片，模擬聊天介面

### Section 6: 功能介紹 — 「三步驟」

- 步驟 1：告訴它你要寫什麼（AI 研究市場 + 競品）
- 步驟 2：10 分鐘生成完整文章
- 步驟 3：一鍵發布到 WordPress
- 每步搭配小美旁白（斜體灰色）
- 視覺：垂直排列，產品介面模糊截圖

### Section 7: 價格 — 「一篇不到一杯咖啡的錢」

- 小美算帳情境：6 萬 → 3,000，省下的錢夠吃一年下午茶
- 方案卡片用情境標題：「剛開始嘗試」/「認真經營」/「多人協作」
- 收尾：免費 3 篇，不用綁信用卡

### Section 8: 收尾 — 「你的故事，從這裡開始」

- 焦點從小美轉移到讀者：「也許你正在⋯⋯」排比句
- CTA：「開始我的故事」
- 視覺：深色調 + 底部暖光漸亮，橘金色按鈕

## 視覺設計原則

- 色調：從 cyberpunk 轉為溫暖共感（暖橘、柔金、淡綠）
- Light mode 為主要體驗，dark mode 保留但調性一致
- 大量留白，每句之間有呼吸空間
- 動畫：淡入為主，不花俏
- 字體：維持 Inter，但善用字重和大小營造節奏

## 技術實作

### 修改檔案

- `src/app/home-client.tsx` — 主要重寫
- `src/components/home/` — 新增/修改 section 元件
- `src/messages/zh-TW.json` — 新增故事文案
- `src/messages/en-US.json` — 英文翻譯
- `src/messages/ja-JP.json` — 日文翻譯
- 其餘 4 個語系檔案同步更新

### 保留項目

- Pricing 的資料庫 fetch 邏輯（plans + packages）
- Navbar 和 Footer
- CTA 指向 `/login`
- FAQ Section（內容可微調）

### 移除項目

- 舊版 9 宮格功能區塊
- 舊版 AI 技術展示區
- 舊版統計數據條
- 舊版成本比較表（改為故事化呈現）
